## 快速开始

### 前置准备

#### 1. 获取 API Key

访问以下平台获取 API Key：
- **阿里云百炼**: https://www.aliyun.com/product/dashscope（推荐）
- **OpenAI**: https://platform.openai.com/
- **其他兼容 OpenAI 接口的平台**

#### 2. 配置环境变量

在 `server/` 目录下创建 `.env` 文件：

```env
# AI 模型配置
API_KEY=your_api_key_here
BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MODEL=qwen3.5-flash
EMBEDDING_MODEL=text-embedding-v3

# 服务器配置
FLASK_DEBUG=false
PORT=5000

# Redis 配置（可选，用于状态持久化）
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0

# MCP 配置
MCP_HOST=localhost
MCP_PORT=8080
```

## 启动服务

### 方式一：本地开发模式

#### 1. 启动 MCP 服务（工具服务）

```bash
cd server
pip install -r requirements.txt
python mcp_module/start.py
```

MCP 服务器运行在: `http://localhost:8080/mcp`

#### 2. 启动后端服务

**新开终端**：

```bash
cd server
python app.py
```

后端服务运行在: `http://localhost:5000`

#### 3. 启动前端服务

**新开终端**：

```bash
cd client
npm install
npm run dev
```

前端服务运行在: `http://localhost:5173`

#### 4. 访问应用

- **智能体聊天界面**: http://localhost:5173
- **向量库管理界面**: http://localhost:5173/vector-db

---

### 方式二：Docker 部署

```bash
cd server

```

服务端口映射：

## 功能说明

### 核心功能

1. **多轮对话** - 支持上下文保持的多轮对话
2. **知识库检索（RAG）** - 从多个知识库检索相关信息
3. **工具调用** - 支持天气查询、钉钉日程、表单提交等
4. **技能执行** - 支持绘图、数据分析、旅行规划等技能
5. **意图识别** - 支持多意图识别和分层路由
6. **情绪感知** - 检测用户情绪并动态调整响应

### 界面说明

- **顶部**：显示"智能助手"标题，提供向量数据库入口
- **中间**：聊天内容区域
  - 左侧：助手回复（白色气泡）
  - 右侧：用户消息（紫色渐变气泡）
- **底部**：输入框和发送按钮
  - 支持回车发送
  - 支持点击按钮发送

### 初始状态

进入应用后，会自动显示欢迎消息：
> "您好！我是智能助手，有什么可以帮助您的吗？"

## 自定义配置

### 修改系统提示词

编辑 `server/config/prompt_config.yaml` 文件：

```yaml
system_prompt: |
  你是一个智能助手，你会根据用户问题来回答用户的问题。
  # ... 自定义内容
```

### 添加新工具

1. 在 `server/mcp_module/tools/` 创建工具文件
2. 使用 `@register_tool` 装饰器注册
3. 重启 MCP 服务器

### 添加新技能

1. 在 `server/skills/` 创建技能目录
2. 创建 `SKILL.md` 文件（参考现有技能格式）
3. 重启后端服务（自动加载）

## 常见问题

**Q: 启动时提示 API Key 错误？**
A: 检查 `server/.env` 文件是否正确配置了 `API_KEY` 和 `BASE_URL`

**Q: 向量化失败？**
A: 检查网络连接，确认 API Key 有 embedding 权限，确保 `EMBEDDING_MODEL` 配置正确

**Q: 如何重置知识库？**
A: 删除 `server/db/` 目录下的向量数据文件，重启服务即可自动重建

**Q: MCP 服务器端口被占用？**
A: 使用 `taskkill /F /PID <PID>` (Windows) 或 `kill -9 <PID>` (Linux) 终止占用进程

**Q: 工具调用失败？**
A: 检查 MCP 服务器是否正常运行，确保能访问 `http://localhost:8080/mcp`

**Q: 会话状态丢失？**
A: 确保 Redis 服务正常运行，或切换到 Memory 检查点模式

**Q: 前端无法连接后端？**
A: 检查 CORS 配置，确保后端服务运行在 `http://localhost:5000`
